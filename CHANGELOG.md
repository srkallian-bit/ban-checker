## v1.0.5 — 2026-05-03

### Новая архитектура загрузки

- **Инкрементальная синхронизация** — при запуске сервер проверяет общее количество банов на API, сравнивает с локальной БД и догружает только недостающие страницы. Больше нет полной перезагрузки при каждом запуске
- **Уникальные идентификаторы** — каждый бан получает `_id` (autoincrement), что исключает путаницу при дедупликации. Автоматическая миграция для существующих БД
- **Поллинг новых банов каждую секунду** — сервер самостоятельно проверяет появление новых банов каждые 1 секунду. Новые баны моментально появляются в таблице
- **Защита от параллельных запросов** — pollInFlight флаг не даёт поллингу накладываться друг на друга

### Фронтенд

- **Убрана бесконечная откладка** — удалены pollStatus(), checkReconnect(), startAutoRefresh(). Фронтенд больше не опрашивает /api/bans/status в цикле каждую секунду
- **Простое обновление** — таблица обновляется каждые 5 секунд одним запросом. Сервер сам управляет всей логикой синхронизации

## v1.0.4 — 2026-05-03

### Исправления

- **Баны не загружались** — Cloudflare блокировал TLS-подключения от Node.js (JA3 fingerprinting). Добавлен совместимый набор шифров во все HTTPS-запросы
- **Парсер JSON ломал ответы API** — кастомный посимвольный парсер некорректно обрабатывал экранированные кавычки и вложенные объекты. Заменён на regex + стандартный JSON.parse
- **Упавшие страницы терялись навсегда** — при ошибке загрузки страницы (timeout, 429, 500) она просто пропускалась. Теперь возвращается в очередь с ретраями (до 3 попыток)
- **Приложение зависало после ошибки загрузки** — если первый запрос к API падал, повторная загрузка не запускалась. Добавлен авторетрай через 30 секунд
- **После смены cookie / reload загрузка зависала** — в настройках оставались старые значения total и stats, что ломало логику boot() при рестарте
- **Неправильная статистика после reload** — переменные suspCount/invCount утекали в global scope. Добавлено объявление через let
- **Баны не добирались до 100%** — дубли и пропущенные страницы не давали loadedCount достичь total. Добавлен порог 1% — если не хватает менее 1%, загрузка считается завершённой
- **Бейджи без перечёркивания** — при открытии деталей бана из профиля разбанованные баны с ошибкой показывались без зачёркивания (extraCls не доставался из getBadge)
